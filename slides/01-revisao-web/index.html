Internet ≠ Web

Internet

A infraestrutura física: cabos, roteadores, e os protocolos de transporte (TCP/IP).

Web

Um sistema de informação de hipermídia que roda sobre a Internet: recursos identificados por URIs e ligados por hiperlinks.

A Web é uma das aplicações da Internet — não são sinônimos.
Analogia: Internet = malha rodoviária; Web = um dos serviços que trafega nela (e-mail, streaming e jogos também usam a Internet, mas não são "a Web").

As 3 tecnologias fundadoras (1989–91, CERN)

HTML HTTP URL

Tim Berners-Lee, no CERN, uniu marcação + protocolo + endereçamento. Continuam sendo a base da Web moderna.

Depois vieram CSS (1996), JavaScript (1995), AJAX (2005), HTML5 (2014), WebAssembly (2017) — camadas sobre a mesma fundação.

Cliente ↔ Servidor

Cliente (User Agent)

Quem inicia a conversa e faz a requisição. Navegador, app móvel, curl, um bot.

Servidor

Quem escuta uma porta (80/443), processa e devolve a resposta. Nginx, Apache, seu app Node.

Papéis funcionais, não hardware: um mesmo computador pode ser cliente e servidor ao mesmo tempo.
Reforçar: "servidor" tem dupla acepção — a máquina e o software. Aqui interessa o software.

O navegador é um cliente especial

Seu núcleo é o motor de renderização: recebe HTML/CSS/JS e constrói a árvore DOM para exibir e interagir.

HTTP: a língua da Web

Baseado em texto, sem estado (stateless), orientado a requisição → resposta.

Cliente— requisição → Servidor← resposta — Cliente
Stateless: cada requisição é independente — o servidor não "lembra" da anterior sem ajuda (cookies, tokens, sessão).
Stateless é o que permite escalar horizontalmente. O preço: reenviar identidade a cada requisição — gancho para o Cap. 8 (auth).

Anatomia da requisição

GET /produtos HTTP/1.1
Host: www.exemplo.com
User-Agent: Mozilla/5.0
Accept: text/html

Linha (método + caminho + versão) · Cabeçalhos (metadados) · Corpo (opcional — POST/PUT enviam, GET não).

Mostrar ao vivo no DevTools → aba Network → Headers.

Anatomia da resposta

HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1256
Cache-Control: max-age=3600

Linha de status · Cabeçalhos · Corpo (HTML, JSON, imagem…).

Códigos de status

ClasseSignificadoExemplos
1xxInformacional100
2xxSucesso200, 201
3xxRedirecionamento301, 302
4xxErro do cliente400, 404
5xxErro do servidor500, 503
Regra de ouro: 4xx é culpa do pedido; 5xx é culpa do servidor.

DNS: a lista telefônica da Internet

Humanos leem nomes; a rede roteia por números. O DNS traduz um pelo outro.

Root ↓ TLD (.com) ↓ Autoritativo
Normalmente UDP porta 53 (rápido). A resposta é cacheada por um TTL. TCP entra em respostas grandes / DNSSEC.
Demonstrar: nslookup ou dig exemplo.com.

O ferramental do dev

DevTools

O laboratório dentro do navegador: DOM, CSS ao vivo, Network, Performance, debug de JS, storage. F12.

VS Code

Editor leve e extensível: terminal e Git integrados, IntelliSense, depurador.

DevTools é a ferramenta que mais usaremos para "ver" o HTTP do capítulo. Abrir e passear.

Git & GitHub

Git — controle de versão distribuído: histórico, branches, reverter, colaborar sem perder código.

GitHub — hospeda repositórios, issues, pull requests, Actions e o GitHub Classroom da disciplina.

Ambientes online (CodePen, JSFiddle, StackBlitz) servem para experimentar sem instalar nada.
Amarrar com a avaliação: entregas via GitHub Classroom. Primeira atividade: enviar hello.html.

Resumo

  • Internet é a infraestrutura; a Web roda sobre ela.
  • Tudo é cliente-servidor falando HTTP.
  • HTTP é stateless: requisição → resposta, com status.
  • Uma URL vira página via DNS → IP → conexão → render.
  • DevTools, VS Code e Git/GitHub são o ferramental base.
